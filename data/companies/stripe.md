# Stripe

## Format

- **Stages:** Application → Take-home project OR OA → Technical interview → Onsite (3–5 rounds)
- **Coding tools:** CoderPad or shared editor; take-home in your own environment
- **Take-home:** Stripe often uses a multi-hour take-home coding project (real-world style) rather than LeetCode-style OA
- **Lookups allowed:** Yes for take-home; interview rounds may vary
- **AI usage:** Not explicitly published; assume no unless told otherwise for live rounds

## Focus areas

- Clean, production-quality code — Stripe is known for caring about code quality over raw DSA speed
- API design and error handling
- Financial systems intuition (idempotency, consistency, reliability)
- Debugging and reasoning about edge cases
- Systems design for senior roles; lighter touch for new grad

## Behavioral

Stripe values:
- **Craftsmanship** — caring about code quality and long-term maintainability
- **User empathy** — thinking about the developer experience
- **Curiosity** — interest in how payments and financial infrastructure work
- **Integrity** — reliability in how you work and communicate

Suggested STAR stories:
1. A time you went back and improved code you wrote earlier
2. A time you found a bug in an edge case before it reached users
3. A time you learned a domain (financial, infrastructure, security) to do your job better

## Recommended prep

**Coding:**
- Focus on code quality, not just correctness — write clean, readable code
- Practice: interval merging, graph BFS/DFS, hash maps, linked lists
- Take-home style: practice building a small feature with error handling and tests

**Portfolio:**
- Projects with tests, clean READMEs, and API design examples are strong signals

# Google

## Format

- **Stages:** OA (STEP / Kickstart for interns) → Phone screen → Onsite (4–5 rounds)
- **Intern-specific:** STEP program for 1st/2nd year; standard SWE internship for 3rd year+
- **Coding tools:** Google Docs (no IDE in interviews — practice this!) or CoderPad depending on team
- **Coding style:** Classic LeetCode-style algorithmic problems — closed-book, whiteboard-equivalent, no execution environment. This is the most "traditional DSA grind" company in this repo's coverage.
- **Lookups allowed:** No lookups in live interviews; OA may allow some resources
- **AI usage:** Not permitted in interviews unless explicitly stated

## Focus areas

- DSA depth — Google is known for hard graph, DP, and tree problems
- Big-O analysis — always required, explain time and space complexity
- Code quality — clean, readable code matters even under time pressure
- Scale reasoning — think about large inputs from the start

## Behavioral

Uses **Googleyness and Leadership** behavioral framework:
- **Role-related knowledge** — what you know about your domain
- **Cognitive ability** — how you break down new problems
- **Leadership** — taking initiative, influencing without authority
- **Googleyness** — comfort with ambiguity, collaborative, mission-driven

Suggested STAR stories:
1. A time you led a project without formal authority
2. A time you solved a problem that had no clear answer
3. A time you had to learn something fast and apply it

## Recommended prep

**Coding:**
- Full NeetCode 150 coverage strongly recommended
- Practice in Google Docs (no syntax highlighting — get comfortable)
- Hard problems: graph traversal, advanced DP, interval scheduling

**Interview style:**
- Think aloud — Google interviewers want to hear your reasoning
- Ask clarifying questions before coding
- Discuss edge cases explicitly

# OpenAI

## Format

- **Stages:** Application → Resume screen → OA → Technical phone screen → Onsite (virtual or in-person)
- **Coding tools:** Typically HackerRank for OA; live coding in technical interviews
- **Lookups allowed:** Varies by stage — OA usually closed, live interview may allow docs
- **AI usage:** Not explicitly published as of mid-2025; assume no AI unless told otherwise

## Focus areas

- Python and systems programming
- Standard DSA (graphs, DP, trees are common)
- ML concepts for ML/research-adjacent roles
- API design and scalability for product engineering roles
- Reliability, distributed systems for infrastructure roles

## Behavioral

Core themes:
- **Impact** — what you built and what it changed
- **Collaboration** — working across teams and disciplines
- **Technical depth** — going beyond the surface in technical decisions
- **Safety awareness** — understanding of AI risks and tradeoffs (for technical roles)

Suggested STAR stories:
1. A project where you had to make a fast technical decision
2. A collaboration that required navigating disagreement
3. A time you caught a reliability or safety issue before it shipped

## Recommended prep

**Coding:**
- Standard Blind 75 + NeetCode 150 coverage
- Focus: graphs, DP, intervals, trees
- Timed practice: 40–45 min per medium problem

**Systems:**
- For SWE roles: review system design fundamentals (rate limiting, caching, queues)
- For ML roles: review model training pipeline concepts
